Lecture #6. Chandy Lamport Algo

Channel: FIFO, reliable connection from one
process to the other (one way)

To init a snapshot: a process
- records its own state
- sends a marker message out on all outgoing channels
- starts recording the messages it receives on all
incoming channels

Recieve a marker message: send marker message
to all other processes.
1st ever: Record state and start recording
incoming messages, mark channel of marker
message as empty

NOT 1st ever:
If process has sent or recieved a marker message:
stop recording from that channel, state of
channel in snapshot is the current state of channel
(marker messages not on snapshots)
↳ Chandy-Lamport

Consistent snapshot: If an event e is in the snapshot,
all events which happened before e are also recorded

O(n²) marker messages for a snapshot with n processes

It works with multiple initiators

# Lecture #6: Chandy Lamport Algo

Channel: FIFO, reliable connection from one process to the other (one way)

### To init a snapshot: a process
- records its own state
- sends a marker message out on all outgoing channels
- starts recording the messages it receives on all incoming channels

### Recieve a marker message: send marker message to all other processes

- 1st ever: Record state and start recording incoming messages, mark channel of marker message as empty
- If process has sent or recieved a marker message: stop recording from that channel, state of channel in snapshot is the current state of channel
  - (marker messages not on snapshots) 

Chandy-Lamport
### Consistent snapshot
If an event e is in the snapshot, all events which happened before e are also recorded

$O(n^2)$ marker messages for a snapshot with n processes

It works with multiple initiators
