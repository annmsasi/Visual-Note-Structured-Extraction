## Structured note

# FIFO message delivery

If a process sends message $m_2$ after message $m_1$, then a process delivering both delivers $m_1$ first. (2 messages, same sender and receiver)

## How to implement?

### Sequence numbers (TCP)
- Messages have id and sequence #s.
- Messages are delivered when they are one more than the last sequence number.

### What if a message gets lost?
- It breaks
- This strategy only works if you have reliable delivery (TCP does)

### Acknowledgements (TCP also does this)
- Can only send one once an acknowledgement is received
- Slower, more steps and also a risk of ack being dropped

## Causal Message Delivery

If the send of a message $m_1$ happens before the send of a message $m_2$, then any process delivering both delivers $m_1$ first. ($m_1$ & $m_2$ could have different senders.)

- All FIFO violations are causal delivery violations, (not vice versa)
- vector clocks can ensure causal message delivery
