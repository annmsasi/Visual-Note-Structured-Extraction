# CSE 138 Lecture #5

- When using a logical clock, you must choose the important elements to track
  - Only deliver a message when every message in its causal history has been delivered
  - Overkill for implementing FIFO delivery causal broadcast protocol
  - NOT totally ordered, as concurrent messages can go in any order
    - $[1,0,0,0]$        $[0,0,0,1]$
    - VCs of independent events
    - neither is bigger
  - They could still disagree, which would be problematic

- Totally ordered + causally ordered message delivery is possible, but hard (w/ fault tolerance)

Unicast Message: One sender, one receiver

Broadcast Message: One sender, everyone receives (including sender)

Multicast Message: One sender, multiple receivers (not necessarily everybody)

## Distributed Snapshot

- We need an algorithm, a time of day clock is not enough, no perfectly synchronized
