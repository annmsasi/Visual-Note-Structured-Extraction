## Structured note

# Lamport Diagrams

happens before is useful in system design & debugging

[figure]

- X, R: Message Send Event
- Q, Z: Message Receive Event
- y: Internal Event

### happens-before relation

$e_1 \to e_2$ IF:

- $e_1$ and $e_2$ occur on the same process with $e_1$ first
- $e_1$ is a message send event and $e_2$ is the corresponding receive event

makes $\to$ a transitive relation:

- if there's another event, $e'$, such that $e_1 \to e'$ and $e' \to e_2$, then $e_1 \to e_2$

### Notes

- Violation of causal message delivery: when a Message B caused by Message A arrives before the Message A.
- Latency, unbounded in distributed systems, causes all sorts of problems like this

### Network Models

set of assumptions about network behavior

- Synchronous Network: No message takes longer than $\Delta$ units of time (physical) to be delivered (Not for this class)
